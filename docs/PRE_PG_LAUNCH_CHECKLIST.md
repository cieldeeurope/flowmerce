# 결제 붙기 전 최종 체크리스트

최종 업데이트: 2026-05-12

이 문서는 `PG 연동 직전` 기준으로 플로우머스에서 반드시 점검해야 하는 항목을 정리한 내부 체크리스트입니다.

## 1. 회원 / 세션

- [ ] 일반 유저 로그인 응답에 `accessToken`과 `accessTokenExpiresAt`이 포함된다.
- [ ] 만료 토큰으로 `mypage`, `flowmerce-studio` API 호출 시 `401` 또는 `403`을 반환한다.
- [ ] 프론트가 만료 응답을 받으면 세션을 제거하고 `/login` 동선으로 자연스럽게 유도된다.
- [ ] `customId`를 쿼리나 body로 바꾸더라도 다른 계정 데이터가 조회/수정되지 않는다.

## 2. 구독 / 권한

- [ ] 결제 성공 후 유저의 `plan`, `subscriptionStartAt`, `subscriptionEndAt`가 자동 반영된다.
- [ ] 결제 실패, 취소, 환불, 해지 시 구독 상태가 자동 반영된다.
- [ ] 플랜별 요청 처리량, 사이트 선택 제한, 하이엔드 사이트 제한이 실제 서버 정책과 동일하다.
- [ ] 구독 종료 후 상품 삭제 원칙과 데이터 처리 기준이 문서와 실제 운영 프로세스에 맞게 정리돼 있다.

## 3. 결제 플로우

- [ ] 결제 시작 페이지에서 최종 플랜, 과금 주기, 동의 문서 링크를 다시 확인할 수 있다.
- [ ] 결제 직전 `이용약관`, `개인정보처리방침`, `구독 및 운영 동의서` 동의 여부를 저장한다.
- [ ] 결제 성공 / 실패 / 취소 URL이 각각 분리되어 있다.
- [ ] 웹훅 기반으로 중복 결제, 지연 승인, 실패 복구를 처리할 수 있다.

## 4. 온보딩 / 세팅

- [ ] 결제 후 고객이 무엇을 보내야 하는지 가격 페이지와 안내 문구가 동일하다.
- [ ] 카카오톡 온보딩 시 받아야 하는 항목이 체크리스트로 정리돼 있다.
- [ ] 사이트 확정, 호스팅 계정 저장, 카테고리 로딩, 수집 테스트까지 실제 운영 순서가 문서화돼 있다.
- [ ] 세팅 완료 시점과 서비스 시작 시점이 내부 기준으로 정의돼 있다.

## 5. Studio 실사용

- [ ] `flowmerce-studio`에서 사이트 선택, 매핑, 마진, 치환, 수집 예약, 진행 상태 확인이 모두 동작한다.
- [ ] 고객이 자기 계정 데이터만 보고 수정할 수 있다.
- [ ] Cafe24 / Smartstore / Godomall / Makeshop 연동 흐름이 실제 운영 서버 기준으로 검증돼 있다.
- [ ] 에러 문구가 고객 관점에서 이해 가능하게 정리돼 있다.

## 6. 운영 / 고객 안내

- [ ] 가격 페이지, 가이드 페이지, 푸터 문서 링크의 문구가 서로 충돌하지 않는다.
- [ ] 결제 후 세팅 방식, 환불 기준, 종료 후 상품 삭제 원칙이 모두 공개 문서와 일치한다.
- [ ] 지원 채널, 응답 시간, 세팅 소요 예상치가 내부 운영 기준과 맞는다.
- [ ] 자주 묻는 질문에서 `왜 카카오톡 온보딩인지`, `상품 삭제 원칙`, `계정 승인 방식`을 설명할 수 있다.

## 7. 런칭 직전 권장 검증

- [ ] 새 유저 회원가입 → 승인 → 로그인 → 사이트 선택 → Studio 진입까지 직접 점검
- [ ] 토큰 만료 또는 강제 401 상황에서 자동 로그아웃 점검
- [ ] 구독 종료 유저에서 접근 제한 / 안내 문구 점검
- [ ] 모바일 환경에서 가격 / 가이드 / 문의 / 로그인 흐름 점검
- [ ] 실제 운영용 카카오톡 링크와 문의 링크 점검

## 이번 작업에서 반영한 준비 상태

- 프론트는 일반 유저 세션에 `accessToken`, `expiresAt`을 저장할 수 있게 정리됨
- `flowmerce-studio` 주요 API 호출에 유저 인증 헤더와 만료 시 로그아웃 처리 연결
- 가격 페이지에 운영 기준 / 동의 문서 동선 보강
- 회원가입 전에 필수 동의 항목 확인 UI 추가
- 구독 및 운영 동의서 페이지 추가

## 아직 남는 핵심 완료 조건

- 실제 백엔드 로그인 응답에서 유저 토큰 발급
- 유저 보호 API 전반에서 토큰 검증 및 계정 본인 확인
- PG 연동 후 구독 상태 자동 반영
- 결제 직전 동의 기록 저장
